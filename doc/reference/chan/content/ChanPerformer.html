Class Overview

Provides performing connectivity with chan.

By default you must implement the following methods:

  • onReadThreads(ReadThreadsData)
  • onReadPosts(ReadPostsData)
  • onReadBoards(ReadBoardsData)

Depending one the ChanConfiguration.Board configuration you must implement the following methods:

OptionMethod
allowSearch onReadSearchPosts(ReadSearchPostsData)
allowCatalog onReadThreads(ReadThreadsData) (see description)
allowArchive onReadThreadSummaries(ReadThreadSummariesData)
allowPosting onReadCaptcha(ReadCaptchaData)
allowPosting onSendPost(SendPostData)
allowDeleting onSendDeletePosts(SendDeletePostsData)
allowReporting onSendReportPosts(SendReportPostsData)

If you configure some options, you also must implement the following methods:

OptionMethod
OPTION_READ_SINGLE_POST onReadSinglePost(ReadSinglePostData)
OPTION_READ_POSTS_COUNT onReadPostsCount(ReadPostsCountData)
OPTION_READ_USER_BOARDS onReadUserBoards(ReadUserBoardsData)
OPTION_ALLOW_CAPTCHA_PASS onCheckAuthorization(CheckAuthorizationData)
OPTION_ALLOW_USER_AUTHORIZATION onCheckAuthorization(CheckAuthorizationData)

public ChanPerformer.ReadCaptchaResult onReadCaptcha (ChanPerformer.ReadCaptchaData data)

Calls when application requests chan to read captcha.

This method must be overridden if option allowPosting is enabled for this board.

This method must return ChanPerformer.ReadCaptchaResult with captcha data.

If your chan uses custom captcha, you must specify a resulting image using setImage(Bitmap).

In the case of Yandex Captcha, resulting captchaData must contain challenge string by CHALLENGE. key

In the case of Google reCAPTCHA and Mail.Ru Nocaptcha, resulting captchaData must contain API key by API_KEY. key

If your captcha has short lifetime, you can check mayShowLoadButton. If this argument equals true, the result may hold NEED_LOAD. This will show "Click to load captcha" button for user.
